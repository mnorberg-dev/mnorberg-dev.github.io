The Databricks Tool You Didn't Know You Needed

September 16, 2025 · 4 min · Matthew Norberg

The Databricks Tool You Didn’t Know You Needed#

Databricks is quickly becoming one of the most popular data lakehouse platforms available. Its popularity is growing rapidly. For many developers who use the platform, a good portion of their job is done in a web browswer in their Databricks environment. Yes, I know there are popular extensions that allow you to develop your code and pipelines outside of Databricks. For instance, many developers love the VS Code extension so they can write code in their favorite editor. At the end of the day though, there’s still a portion of time spent in the browser making sure your code works as expected.

Many organizations that use databricks have different workspaces to represent different environments. For example, you might have dev, qa, and prod workspaces. You might also have dev and prod environments as well. Either way, it’s to all those developers who have different databricks workspaces that this blog post is geared towards. If you only have one databricks workspace, who knows, you might find this article useful as well.

So what’s the issue if you spend a bunch of time working in the browswer and you have multpile databricks workspaces to represent different environments? Well, unless you use multiple browsers, then your dev environment looks almost exactly the same as your prod and qa environments. As a developer, you end up with a bunch of different browser tabs open. It get’s hard to remember which one is for which environment. Plus you run the risk of making a change in prod or qa that you meant to make in dev.

So here’s one solution that might work. You assign one environment the light color scheme and another environment the dark color scheme. But there’s only two color options available so at least one of your environments is going to match with another. Plus, let’s say two of your environments are in the same region, and thus share a metastore. In that case, the settings in each environmentt are shared. For example, suppose your dev and qa are in the same region. If you change your dev color scheme, your qa color scheme will change as well. Not to mention, I know many devs that cringe at the idea of using a light color scheme and vice versa. So let’s find a solution to differentiate our environments and keep your color scheme of choice.

The Solution?

There is a popular chrome extension called Tamper Monkey which allows you to writet custom JavaScript code that executes when your navigate to a url that matches the url in the script. Using this extension, we can edit the title of our browswer tabs to display a custom message identifying the environment you are in.

This solves the problem of identifying which workspace corresponds to which environment. It improves your quality of life as a developer and improves your productivity. It also reduces the chances of making a change in an unintended environment.

Also, this solution is lightweight enough so that it does not degrade the performance of your browswer. I have not noticed any difference in browser speed. However, after using this tool I made for a week, I have no idea how I lived without it.

How’s it work#

I’ve posted the script on github linked here but I will provide it in this article with some instructions on how to set it up. You can find additional instructions with more details in the README file if you need it.

// ==UserScript==
// @name         Databricks Tab Emoji Label
// @namespace    http://tampermonkey.net/
// @version      1.5
// @description  Add emoji and label to Databricks tab title based on environment
// @author       Matthew Norberg
// @match        https://<your-domain-here>.azuredatabricks.net/*
// @match        https://<your-domain-here>.azuredatabricks.net/*
// @match        https://<your-domain-here>.azuredatabricks.net/*
// @grant        none
// ==/UserScript==

(function() {
    'use strict';

    function updateTitle() {
        const url = window.location.href;
        let label = '';
        let emoji = '';

        // Define your environment domains here
        let devDomain = '';
        let qaDomain = '';
        let prodDomain = '';

        if (url.includes(devDomain)) {
            label = 'DEV';
            emoji = '🟢';
        } else if (url.includes(qaDomain)) {
            label = 'QA';
            emoji = '🟡';
        } else if (url.includes(prodDomain)) {
            label = 'PROD';
            emoji = '🔴';
        } else {
            label = 'OTHER';
            emoji = '⚪';
        }

        if (!document.title.startsWith(`[${emoji} ${label}]`)) {
            document.title = `[${emoji} ${label}] ${document.title}`;
        }
    }

    // Initial run
    setTimeout(updateTitle, 3000);

    // Monitor for URL changes in SPA
    let lastUrl = location.href;
    new MutationObserver(() => {
        const currentUrl = location.href;
        if (currentUrl !== lastUrl) {
            lastUrl = currentUrl;
            setTimeout(updateTitle, 2000); // Slight delay to allow page content to load
        }
    }).observe(document, {subtree: true, childList: true});

})();
